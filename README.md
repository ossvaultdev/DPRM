# DATA PRODUCT REFINED MANAGEMENT
# DPRM PROJECT 

## 1. Description: 
This is a web server allows the user to generate Web API requests and then schedule them at user-defined intervals. It then filters and transforms the data for use with an API server or for data extraction in various formats, such as Text, JSON, or XML files, or into a database for their applications use.

![DPRM Diagram](https://github.com/ossvaultdev/DPRM/blob/main/web/static/image/diagram_process.jpg)

## 2. Design: 
The user will need to authenticate with a login after registering and will then have access to their workspaces, which are defined in three steps as described below.
## 2.1 Acquisition: Web Page I
This web page will allow the user to create one or more web API requests. On this page, there will be a function to launch the web API request(s) and receive data in lists on the same page. 

From this page, the user can also schedule web API requests to be automatically executed at user-defined intervals.
## 2.2 Transformation: Web Page II
This web page will display the results from the Acquisition page "Web Page I" in one or more data tables so that the user can transform them, modifying the data by calculating, adding, cross-referencing with other data, and reducing it to obtain the necessary dataset for their applications.
## 2.3 Data Processing: Web Page III
This page allows users to specify and configure how to process the acquired and transformed data. 

This includes the ability to extract files via internet links to download them in various formats, as described in the description, or to share them with other users via internet links or web API requests.

To access the Transformation and Data Processing web pages, the previous step must be completed.
## 3. Development:
For the development of this project, a web server will be needed for the user interface, and an application for managing the web server will be located on the same server for security purpose.
## 3.1 Web Server:
The web server will have a central database dedicated to managing the website, including user creation and authentication to grant access to their workspaces. 

Each user will then have their own database in a specific, dedicated directory on the server. This will be a user-specific database containing all the parameters for each stage of their workspace, as well as the acquired and transformed data.
## 3.2 Application Software:
Having an application for server management increases security by restricting access to sensitive server and user data. It will prevent unauthorized access to the web server administration via a simple internet connection and compromised administrator credentials. 

The application will be located in the same place as the central database on the web server and it will allow the server to be configured and user usage to be managed.
## 4. Market:
There are two possible business strategies to achieve a return on investment with this server and its application.

The first is to sell the web server with its management application to individuals so they can install it on their own system and use it within a closed, private network to ensure the confidentiality of their data. The selling price will be higher than in the second option.

The second option is to have a dedicated server available to the general public with free access initially, but with limited usage and data storage. Later, users could be billed based on the amount of data stored on the server and the frequency of requests generated by their workspaces. This price per user would be lower than the cost of purchasing the software.
